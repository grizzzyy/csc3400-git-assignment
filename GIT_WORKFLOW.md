GIT WORKFLOW 

1) Branching strategy 
   main is the stable branch 
   new work is done on feature branches like /feature, after the work is done it is merged into main

2) Commit message 
   short, clear messages
   
3) Code review 
   create a pull request on github
   add description of what changed
   review changes before merging

4) Release workflow
   merge completed branches into main
   tag the release version 
   push to github

5) Common git commands
   git status
   git add .
   git commit -m "message"
   git checkout -b branch name
   git pull
   git push
